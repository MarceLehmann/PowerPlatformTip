This causes broken HTTP requests, failed API calls, invalid JSON errors, and runtime failures in both Power Apps and Power Automate. Without proper encoding, URLs with query parameters break, filter queries fail, and data transmission produces unpredictable results.\n\n## \u2705 Solution\n\nUse native encode/decode functions to convert special characters into safe escape sequences (%XX format). Power Apps offers EncodeUrl() and EncodeHTML() for encoding output, but lacks native decoding. Power Automate provides full bidirectional support through encodeUriComponent() and decodeUriComponent(), essential for HTTP requests and JSON payloads. Important: OData filter queries require different escaping techniques.\n\n## \ud83d\udd27 How It's Done\n\n### Power Apps Functions\n\n**1. EncodeUrl() \u2013 encode URLs and query parameters safely**\n\n\ud83d\udd38 Expression: `EncodeUrl(\"https://example.com/search?q=hello world\")`\n\n\ud83d\udd38 Returns: `\"https://example.com/search?q=hello%20world\"`\n\n\ud83d\udd38 Special characters: Space\u2192%20, ?\u2192%3F, &\u2192%26, #\u2192%23\n\n\ud83d\udd38 Use case: `Launch(EncodeUrl(\"https://wa.me/?text=Check O'Neill's offer!\"))`\n\n**2. EncodeHTML() \u2013 prevent HTML injection and display issues**\n\n\ud83d\udd38 Expression: `EncodeHTML(\"<script>alert('test')</script>\")`\n\n\ud83d\udd38 Returns: `\"&lt;script&gt;alert('test')&lt;/script&gt;\"`\n\n\ud83d\udd38 Converts: < \u2192 &lt;, > \u2192 &gt;, & \u2192 &amp;, \" \u2192 &quot;\n\n\ud83d\udd38 Use case: Display user comments safely in HTML Text controls without XSS vulnerability\n\n**3. PlainText() \u2013 strip HTML tags and decode entities**\n\n\ud83d\udd38 Expression: `PlainText(\"<p>Hello &nbsp; &quot;World&quot;</p>\")`\n\n\ud83d\udd38 Returns: `\"Hello   \"World\"\"`\n\n\ud83d\udd38 Removes all HTML/XML tags and converts entities to readable text\n\n\ud83d\udd38 Use case: Clean SharePoint rich text fields for plain text display or search\n\n**4. DecodeUrl() limitation**\n\n\ud83d\udd38 No native function exists in Power Apps\n\n\ud83d\udd38 Solution: Call a Power Automate flow with the decodeUriComponent() expression\n\n### Power Automate Expressions\n\n**5. encodeUriComponent() \u2013 critical for URL parameters**\n\n\ud83d\udd38 Expression: `encodeUriComponent('How are you?')`\n\n\ud83d\udd38 Returns: `'How%20are%20you%3F'`\n\n\ud83d\udd38 Special characters: Space\u2192%20, ?\u2192%3F, '\u2192%27, \"\u2192%22, {\u2192%7B, }\u2192%7D\n\n\ud83d\udd38 Use case 1: Encode JSON as URL parameter: `encodeUriComponent('{\"name\":\"John\"}')`\n\n\ud83d\udd38 Use case 2: Prepare query string parameters for HTTP GET requests\n\n**6. decodeUriComponent() \u2013 convert encoded data back to readable format**\n\n\ud83d\udd38 Expression: `decodeUriComponent('https%3A%2F%2Fexample.com%2Fsearch%3Fq%3Dhello%20world')`\n\n\ud83d\udd38 Returns: `'https://example.com/search?q=hello world'`\n\n\ud83d\udd38 Decode apostrophes: `decodeUriComponent('Tom%27s%20Diner')` returns `'Tom's Diner'`\n\n\ud83d\udd38 Use case: Process encoded webhook data or query string parameters from external systems\n\n**7. JSON escaping for HTTP request body**\n\n\ud83d\udd38 Problem: Double quotes in JSON break HTTP request syntax\n\n\ud83d\udd38 Use case (Request Body): Use proper JSON escaping with replace()\n\n\ud83d\udd38 Expression: `replace(string(variables('jsonData')), '\"', '\\\"')`\n\n\ud83d\udd38 Result: Escapes double quotes with backslash so JSON remains valid\n\n\ud83d\udd38 Alternative: Use backslash escape: `replace(body('Get_item')?['Title'],'\"','\\\"')`\n\n**8. OData filter query with apostrophes**\n\n\ud83d\udd38 Problem: Filter query fails with `Title eq 'Tom's Diner'`\n\n\ud83d\udd38 Solution: Use replace() to double apostrophes: `replace(Title,'''','''''')`\n\n\ud83d\udd38 How it works: Two apostrophes (`''`) represent one apostrophe in the data\n\n\ud83d\udd38 Result: `'Tom's Diner'` becomes `'Tom''s Diner'` in the OData query\n\n\ud83d\udd38 Important: Do NOT use encodeUriComponent() for OData filters - it breaks the query syntax\n\n## \ud83c\udf89 Result\n\nDevelopers can safely transmit data containing quotes, apostrophes, JSON structures, and special characters without syntax errors or runtime failures. HTTP requests work reliably, OData queries process correctly with proper apostrophe escaping, and the distinction between code and content is preserved across Power Platform solutions.\n\n## \ud83c\udf1f Key Advantages\n\n\ud83d\udd38 Prevents \"invalid JSON\" errors in HTTP actions through proper quote escaping\n\n\ud83d\udd38 Enables safe transmission of JSON objects as URL parameters with encodeUriComponent()\n\n\ud83d\udd38 Solves OData filter query failures when values contain apostrophes using replace()\n\n\ud83d\udd38 Distinguishes between syntax characters and data content in Power Fx and expressions\n\n\ud83d\udd38 No premium connectors required, works with standard HTTP and SharePoint actions\n\n---\n\n## \ud83d\udee0\ufe0f FAQ\n\n**Q1: Why do my HTTP requests fail with \"invalid JSON\" even though the JSON looks correct?**\n\nThe issue is often unescaped double quotes in dynamic values that break JSON syntax. For URL parameters, use encodeUriComponent(). For JSON request bodies, use replace() to escape quotes: `replace(variables('text'), '\"', '\\\"')`. Remember: quotes have special meaning in both JSON and Power Automate expressions.\n\n**Q2: How do I handle text with apostrophes in Power Automate OData filter queries?**\n\nUse the replace() expression to double apostrophes: `replace(varTitle,'''','''''')`. This escapes the apostrophe so `'Tom's Diner'` becomes `'Tom''s Diner'` in the query. Two apostrophes together are interpreted as a single apostrophe in the data. NEVER use encodeUriComponent() for OData filters as it encodes the filter syntax itself and breaks the query.\n\n**Q3: What's the difference between encodeUriComponent() for URLs vs. JSON escaping for request bodies?**\n\nUse encodeUriComponent() only for URL parameters and query strings - it converts characters to %XX format. For JSON in HTTP request bodies, use replace() with backslash escaping: `replace(text, '\"', '\\\"')`. URL encoding and JSON escaping serve different purposes and are not interchangeable.\n

Need a smaller version of an image inside a flow — without a premium image-processing connector or external editor?
SharePoint already generates **thumbnails** for images stored in a library. You can request a specific size and save that resized copy back with **Create file**.

## 💡 Challenge
Large images are awkward to handle when you only need a smaller version, and full image-editing actions often mean premium connectors or extra tooling.

## ✅ Solution
Store the image in SharePoint and pull a resized rendition from SharePoint's built-in thumbnail endpoint using **Send an HTTP request to SharePoint**, then save the result.

## 🔧 How it's done

**1. Store the image in SharePoint**

🔸 Upload (or already have) the original image in a document library.

**2. Request a resized thumbnail**

🔸 Use **Send an HTTP request to SharePoint** (Method `GET`) against the preview endpoint, specifying the width and height you want:

```
_layouts/15/getpreview.ashx?path=/sites/YourSite/Shared Documents/image.jpg&width=400&height=400
```

🔸 SharePoint returns a resized rendering of the image at (or within) the requested dimensions.

**3. Save the smaller image**

🔸 Pass the response body into a **Create file** action to store the resized copy wherever you need it.

## 🎉 Result
You get a smaller image generated by SharePoint itself — no external editor and no premium image connector required.

## 🌟 Key Advantages

🔸 **No extra tooling:** SharePoint does the resizing for you.

🔸 **No premium connector:** uses the standard SharePoint action.

🔸 **Flexible sizes:** request the width and height you need per call.

---

## 🛠️ FAQ

**Q1: Which image formats does this work with?**

Common web image formats such as JPEG and PNG render reliably through SharePoint's thumbnail service.

**Q2: Can I resize multiple images in one run?**

Yes. Loop with **Apply to each** over your images, mindful of the flow's execution-time limits.

**Q3: Does the thumbnail keep the original aspect ratio?**

SharePoint fits the image within the requested width and height while preserving the aspect ratio, so the result may be smaller than the exact numbers you pass. Avoid upscaling beyond the original size.
